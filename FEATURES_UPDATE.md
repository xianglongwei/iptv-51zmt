## 修改内容

### 1. 手动触发爬虫功能 (用户问题 2)

**问题**: 自动爬虫功能存在但缺少手动触发按钮。

**解决方案**:

#### 后端修改 (`backend/main.py`)
- 添加了新的API端点 `POST /api/crawler/run`
- 该端点会：
  1. 执行 `iptv.py` 爬虫脚本
  2. 等待脚本完成（60秒超时）
  3. 自动导入更新后的 `2.m3u` 文件到数据库
  4. 返回导入统计信息（新增、更新数量）

#### 前端修改 (`frontend/index.html`)
- 在顶部导航栏添加了绿色的"手动爬取"按钮
- 按钮状态：
  - 默认显示"手动爬取"
  - 爬取中显示"爬取中..."并禁用按钮
- 功能：
  - 点击后调用 `/api/crawler/run` API
  - 完成后自动刷新频道列表
  - 显示操作结果（新增/更新数量）

## 使用说明

### 手动爬虫功能
1. 在前端界面顶部找到绿色的"手动爬取"按钮
2. 点击按钮，系统会自动：
   - 运行 `iptv.py` 脚本抓取最新频道数据
   - 更新 `2.m3u` 文件
   - 导入新数据到数据库
3. 等待完成后会显示爬取结果
4. 频道列表会自动刷新

### 预览功能调试
1. 点击任意频道的"诊断"按钮
2. 在诊断面板中点击"播放预览"
3. 查看诊断日志和控制台（F12）获取详细信息：
   - 频道名称和URL
   - 检测到的播放模式
   - 代理地址（如使用MPEG-TS）
   - 错误信息（如有）

## 技术细节

### API端点：`POST /api/crawler/run`
- **请求**: 无需参数
- **响应**:
  ```json
  {
    "status": "success",
    "message": "Crawler completed successfully",
    "import_stats": {
      "added": 5,
      "updated": 10,
      "unchanged": 15
    },
    "timestamp": "2026-03-29T14:30:00"
  }
  ```
- **错误处理**:
  - 60秒超时: 返回504错误
  - 脚本执行失败: 返回500错误及详细信息

### 预览模式检测
前端会自动检测流URL类型并选择合适的播放器：
- **HLS** (`.m3u8`): 使用hls.js播放器
- **MPEG-TS** (HTTP流): 使用mpegts.js播放器，通过后端代理
- **原生** (`.mp4`, `.webm`等): 使用浏览器原生播放器
- **不支持** (RTSP/RTP/UDP): 显示错误提示

## 注意事项

1. **爬虫运行时间**: 手动爬取可能需要几秒到几十秒，请耐心等待
2. **预览浏览器兼容性**: 
   - 推荐使用Chrome/Firefox/Edge最新版
   - MPEG-TS播放器需要浏览器支持WebAssembly
3. **网络连接**: 预览功能需要后端服务正常运行
