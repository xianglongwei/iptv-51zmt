# 更新总结

## 用户的两个问题

### 问题1: Preview不显示
**原因分析**:
- Preview功能代码存在，后端API端点正常（`/api/preview/stream`）
- 可能是用户未正确触发预览，或者存在浏览器兼容性问题
- 缺少调试信息来诊断具体问题

**解决方案**:
- 在前端 `startPreview` 函数中添加了详细的控制台日志
- 在诊断日志中显示预览启动信息
- 添加代理URL的记录（用于MPEG-TS播放）

### 问题2: 自动爬虫功能在哪里
**原因分析**:
- 自动爬虫功能存在于后端 `scheduler.py`，每12小时自动运行
- 从 `2.m3u` 文件导入数据
- **缺少手动触发按钮**，用户无法立即执行爬虫

**解决方案**:
- 在后端添加了新的API端点：`POST /api/crawler/run`
- 在前端添加了绿色的"手动爬取"按钮
- 按钮会执行 `iptv.py` 脚本并自动导入结果

## 使用指南

### 手动爬虫功能
1. **位置**: 页面顶部导航栏，绿色"手动爬取"按钮
2. **操作**: 点击按钮，等待完成
3. **反馈**: 
   - 爬取中：按钮显示"爬取中..."并禁用
   - 完成：显示新增/更新数量，自动刷新列表

### Preview调试
1. **打开诊断**: 点击任意频道的"诊断"按钮
2. **启动预览**: 点击诊断面板中的"播放预览"
3. **查看信息**:
   - **诊断日志**: 预览模式、代理地址等
   - **浏览器控制台** (F12): 详细调试信息
   - **播放状态**: 加载、播放或错误状态

## 预览模式说明

系统会自动检测URL类型并选择播放器：

| URL类型 | 播放器 | 说明 |
|---------|--------|------|
| `.m3u8` | HLS.js | HTTP Live Streaming |
| HTTP流 | MPEG-TS | 通过后端代理播放 |
| `.mp4`, `.webm` | 原生 | 浏览器原生支持 |
| RTSP/RTP/UDP | 不支持 | 显示错误提示 |

## 注意事项

1. **服务器重启**: 后端代码修改需要重启服务器才能生效
2. **浏览器缓存**: 前端修改可能需要清除缓存或强制刷新（Ctrl+F5）
3. **爬虫时间**: 手动爬取可能需要几秒到几十秒
4. **网络要求**: Preview功能需要稳定的网络连接
